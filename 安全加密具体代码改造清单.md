### `backend/.env.example`

- 新增：
  - `FIELD_ENCRYPTION_KEY=`

## 2. 依赖层

### `backend/requirements.txt`

- 新增依赖：
  - `cryptography`

## 3. 新增加密服务

### `backend/app/services/encryption_service.py`

新增能力：

- `is_encryption_enabled()`
- `_get_fernet()`
- `is_encrypted_value()`
- `encrypt_text()`
- `decrypt_text()`

设计要点：

- 使用 `enc::` 作为密文前缀
- 兼容旧明文数据
- 未配置密钥时允许系统继续运行，但不启用字段加密

## 4. 新增错误脱敏工具

### `backend/app/utils/error_utils.py`

新增：

- `summarize_http_error(prefix, exc)`

作用：

- 提取 `status_code`
- 尝试提取 `request_id`
- 避免完整第三方报文落库或直接暴露给前端

## 5. 任务创建与查询链路改造

### `backend/app/api/routes_jobs.py`

改造点：

- 创建任务时，对 `source_text` 加密后再入库
- 返回任务详情时，对以下字段解密：
  - `source_text`
  - `translated_text`
  - `error_message`
  - `voice_outputs.error_message`
- 翻译阶段先解密原文，再调用翻译服务
- 语音生成结果中的错误信息加密落库
- 全任务失败或部分失败时，任务错误信息加密落库
- 捕获 `ExternalServiceError`，避免未处理异常中断后台任务

## 6. 历史记录链路改造

### `backend/app/api/routes_history.py`

改造点：

- 不再直接对数据库密文字段做 `LIKE`
- 改为先读取任务列表
- 解密 `source_text` 与 `translated_text`
- 在 Python 内存中执行关键字过滤

## 7. 第三方服务调用改造

### `backend/app/services/translation_service.py`

改造点：

- 失败时使用 `summarize_http_error()`
- 不再拼接第三方原始响应全文

### `backend/app/services/elevenlabs_service.py`

改造点：

- 音色列表接口失败时返回脱敏错误
- TTS 接口失败时返回脱敏错误
- 避免直接暴露第三方错误全文

## 8. 传输安全改造

### `backend/app/main.py`

新增：

- `HTTPSRedirectMiddleware`
- CORS 白名单补充本地地址
- 安全响应头中间件：
  - `Strict-Transport-Security`
  - `X-Frame-Options`
  - `X-Content-Type-Options`
  - `Referrer-Policy`

## 9. 历史数据迁移脚本

### `backend/scripts/migrate_encrypt_existing_jobs.py`

用途：

- 扫描数据库已有明文字段
- 自动加密后写回
- 已经加密的数据跳过

新增执行方式：

```bash
cd backend
python -m scripts.migrate_encrypt_existing_jobs
```
